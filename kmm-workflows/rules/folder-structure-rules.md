# Folder Structure Rules

> **Why**: Consistent structure enables quick navigation.

## ✅ DO

- Screens → `modules/{feature}/ui/`
- Components → `modules/{feature}/components/`
- Shared UI → `uiToolKit/`
- ViewModels → `modules/{feature}/viewModel/`
- Repositories → `repository/`
- Models → `model/data/`

## ❌ DON'T

- Don't mix screens and components
- Don't place feature code outside its module
